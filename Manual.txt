The data is stored in an output file called "Database.txt"
Each time the program runs, this data is loaded, and when the program ends, the updated data is overwritten
Incase the program fails to load the file, an error message will be shown as well as if it fails to write it before ending the program

Each product is stored in a separate line, separated by a space, in the format:
Name Type Price Quantity Daysleftbeforexpiry Batch
where name is a string without any space
type is a string without space
price is a double type
quantity is an integer
daysleftforexpiry is an integer
batch is a string without a space
e.g 
Mangoes Perishables 5 1000 3 D1
It is assumed that there is always atleast 1 product in the database and It is not completely empty.

The user will use an interface to interact with the program, with 8 options provided. It can handle incorrect input case, such as a string input instead of a number.
1. Check reminder: 
There will be a reminder for expiry if "Perishables" are about to expire in 1 day.
There will be a reminder for expiry if "Non Perishables" are about to expire within 1 week.
There will be a reminder for excess inventory if there are more than 10000 units of a particular Product.
If there are no reminders, an output message will be displayed stating as such.

2. Add a new product: To add a new entry into the database. Detail by detail the user need to input everything manually.

3. Update existing Product: It lets you update any of the existing products. It asks for which product to update and the batch number to locate the specific product.
In case the product does not exist then an output message will be displayed, otherwise it will prompt the user for the new price, quantity, and expiry date to update the particular product.

4. View Products: You can check the specific information on products, and inspect the whole database using filters such as batch number, quantity, days left, etc.

5. Inventory checking and predicting: Mainly designed for marketing porpose, has some pretty sophsticated embedded function to provide the user with more options and possibly more accurate prediction depends on how much data the user has.

6. Inventory worth: You can check how much money worth of products are in your inventory, you can even check it by products.

7. Inventory checking and predicting in events of natural disaster: An inventory prediction system taken into consideration the natural disaster situation, ask for some indexes to make more accurate prediction for the user.

8. Quit: The updated database will be stored into the file and the program will end.
